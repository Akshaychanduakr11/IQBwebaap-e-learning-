01. ML Paradigms

Machine Learning is the study of algorithms that improve through experience. We categorize ML into three primary types:

Type Core Concept Algorithms
Supervised Learning with Labeled Data Linear Regression, SVM, Random Forest
Unsupervised Finding Hidden Patterns K-Means, PCA, Association Rules
Reinforcement Learning via Reward/Penalty Q-Learning, SARSA

02. Linear & Logistic Regression

These are the foundational blocks of predictive modeling. Linear Regression predicts continuous values, while Logistic Regression handles binary classification.

# Logistic Regression for Binary Classification
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

model = LogisticRegression()
model.fit(X_train, y_train)
predictions = model.predict(X_test)
print(f"Accuracy: {accuracy_score(y_test, predictions)}")

03. Feature Engineering & Selection

A model is only as good as the data it receives. Preprocessing is where 80% of ML work happens.

  • Scaling: Normalization (0-1) and Standardization (Mean=0, Std=1) to ensure features contribute equally.
  • Handling Missing Data: Using Mean/Median imputation or dropping NaN values.
  • Categorical Encoding: Converting text labels into numbers using One-Hot Encoding.

04. Random Forest & Boosting

Individual models can be weak. Ensemble Methods combine multiple models to create a robust "Strong Learner."

  • Bagging (Random Forest): Parallel training on different data subsets.
  • Boosting (XGBoost/AdaBoost): Sequential training where each model corrects the errors of the previous one.

05. Performance Metrics

How do we know if our model is successful? We use specific metrics depending on the problem type:

Classification: Precision, Recall, F1-Score, and Confusion Matrix.
Regression: Mean Squared Error (MSE), R-Squared ($R^2$), and MAE.

Dimensionality Reduction & Hyperparameter Optimization

01. Principal Component Analysis (PCA)

In datasets with hundreds of features, we face the Curse of Dimensionality. PCA reduces complexity by projecting data onto its most significant "Principal Components" while retaining maximum variance.

# Reducing feature space to 2 dimensions
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
X_reduced = pca.fit_transform(X_large_dataset)

02. Clustering: K-Means & Hierarchical

Clustering groups data points without labels. This is essential for Customer Segmentation and Anomaly Detection.

K-Means

Uses centroids and iterative distance minimization.

DBSCAN

Density-based; excellent for identifying clusters of arbitrary shapes.

03. Regularization: Lasso & Ridge

To prevent Overfitting, we penalize large coefficients in our mathematical model. This forces the model to stay simple and generalize better to new data.

  • L1 (Lasso): Adds absolute value of coefficients. Can perform Feature Selection by shrinking useless feature weights to zero.
  • L2 (Ridge): Adds squared value of coefficients. Best for handling Multicollinearity.

04. Optimization (Grid & Random Search)

Machine Learning models have "knobs" called hyperparameters (e.g., Learning Rate, Depth of Tree). We use automated searches to find the best configuration.

# Finding the best parameters for a model
from sklearn.model_selection import GridSearchCV
params = {'n_estimators': [50, 100, 200], 'max_depth': [3, 5, 10]}
search = GridSearchCV(RandomForestClassifier(), params, cv=5)

05. Engineering Interview Topics

Q1: What is Cross-Validation (K-Fold)?

A technique where the data is split into $K$ parts. The model is trained on $K-1$ parts and tested on the remaining part. This process repeats $K$ times to ensure the model isn't just "lucky" with a specific data split.

Q2: Explain the Elbow Method in Clustering?

It is used to find the optimal number of clusters ($K$). We plot the "Sum of Squared Errors" (SSE) against $K$. The point where the rate of decrease slows down (the "elbow") is the ideal $K$.
